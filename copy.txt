  4. 0.1 CONSULT COACH (상담 코치 — 3 stage 세로 스크롤)
═══════════════════════════════════════════════════════════════════
  파일: components/sections/FeatureConsultCoach.tsx
  카피: COPY.consultCoach

  섹션 라벨: [ 0.1 ] CONSULT COACH

  ───────────────────────────────────────────────────────────────
  STAGE 1 (텍스트 좌·이미지 우)
  ───────────────────────────────────────────────────────────────

  캡션 뱃지: 1차 · 온라인 상담 / 전화·카톡

  헤드 (2줄):
    전화 한 통 이후,
    →
    환자분 맞춤 상담 준비가 시작됩니다

  본문 (단락 5개, ⏎⏎ 분리, 단락 내 → 줄바꿈):
    전화 문의와 카카오톡 상담이 쌓일수록
    →
    환자분의 니즈를 빠르게 파악하고, 병원 톤에 맞게 응대하는 일은 쉽지 않습니다.
    ⏎⏎
    특히 새로 합류한 상담 실장님은
    →
    병원의 상담 방식과 대표원장님의 진료 철학을 익히기까지 시간이 필요합니다.
    ⏎⏎
    이음은 카카오톡 상담 대화와 전화 상담 기록을 바탕으로
    →
    환자분의 핵심 요구를 정리하고, 병원에 맞는 상담 가이드라인을 제공합니다.
    ⏎⏎
    베테랑 실장님의 상담 노하우와 병원의 브랜드 톤을 반영해
    →
    누가 상담하더라도 일관된 응대가 가능하도록 돕습니다.
    ⏎⏎
    처음 상담을 시작하는 실장님도
    →
    환자분의 상황을 빠르게 이해하고, 더 안정적인 상담 흐름을 만들 수 있습니다.

  체크리스트:
    ✓ 환자분 핵심 니즈 한 줄 요약
    ✓ 병원 톤을 반영한 권장 상담 멘트 제안
    ✓ 의료광고법과 브랜드 톤을 고려한 주의 표현 안내
    ✓ 콜센터 통화 패턴 분석 · 베타 운영 중

  ───────────────────────────────────────────────────────────────
  STAGE 2 (텍스트 우·이미지 좌, 지그재그)
  ───────────────────────────────────────────────────────────────

  캡션 뱃지: 2차 · 오프라인 현장 / 내원 당일

  헤드 (2줄):
    내원한 환자분의 고민,
    →
    상담 전에 이미 정리되어 있습니다

  본문 (단락 5개, ⏎⏎ 분리, 단락 내 → 줄바꿈):
    병원에서 환자분과 대면 상담할 때,
    →
    환자분들의 가려운 부분을 시원하게 긁어 줄 수 있도록 베테랑 실장님의 노하우가 담긴 스크립트가 작성돼요.
    ⏎⏎
    언제든지 그리고 남녀노소 다양한 연령층의 환자가 방문하여도 간단한 설문만으로 전환율을 높일 수 있는 상담 스크립트가 작성돼요.
    ⏎⏎
    설문이 끝난 환자분이 의자에 앉기 전에,
    →
    이미 정리된 그분의 고민으로 현장 상담이 이어져요.
    ⏎⏎
    "잠깐 이야기했을 뿐인데 나를 이해하고 공감해주네." — 처음 방문한 환자분에게 이 느낌이 가장 중요해요.
    ⏎⏎
    실장님이 차트에 정리만 하는 게 아니라,
    →
    환자분과 마주보고 진정성 있는 대화를 나누실 수 있도록 도와드려요.

  체크리스트 (4개):
    ✓ 1차 상담 내용 자동 인계
    ✓ 현장 상담 내용 정리
    ✓ 상담 기록 기반 차트 초안 작성
    ✓ 원장님 인계용 환자분 핵심 요약

  ───────────────────────────────────────────────────────────────
  STAGE 3 (텍스트 좌·이미지 우)
  ───────────────────────────────────────────────────────────────

  캡션 뱃지: 3차 · 원장님 상담 / 진료실

  헤드 (2줄):
    상담 스크립트부터 비교 케이스까지,
    →
    원장님 상담 자료가 한 화면에 준비됩니다

  본문 (단락 4개, ⏎⏎ 분리, 단락 내 → 줄바꿈):
    1차 문의와 2차 내원 상담에서 나온 내용을 바탕으로
    →
    환자별 진료 브리핑을 자동으로 정리합니다.
    ⏎⏎
    환자분의 고민, 예산, 회복 일정, 원하는 결과를 한눈에 확인하고
    →
    원장님이 바로 참고할 수 있는 상담 스크립트까지 제공합니다.
    ⏎⏎
    환자가 가져온 희망 결과 사진과
    →
    원장님 추천 케이스를 함께 비교해 보여줄 수 있어
    →
    상담 중 필요한 설명을 더 빠르게 이어갈 수 있습니다.
    ⏎⏎
    원장님은 자료를 찾는 시간 없이
    →
    환자분에게 맞는 설명과 추천 방향에 집중할 수 있습니다.

  체크리스트 (4개):
    ✓ 1·2차 상담 기반 진료 브리핑 자동 생성
    ✓ 원장님 참고용 상담 스크립트 제공
    ✓ 환자 희망 사진과 추천 케이스 비교
    ✓ 상담·후기·케어 통합 타임라인

    ✓ 상담 경험 기반 후기 질문 자동 구성
    ✓ 환자분이 쉽게 고를 수 있는 객관식 답변 제공
    ✓ 말투 선택에 맞춘 후기 초안 자동 완성
    ✓ QR 한 번으로 카카오 · 네이버 · 강남언니 후기 작성 바로 연결

  9. FAQ (자주 묻는 질문)
═══════════════════════════════════════════════════════════════════
  파일: components/sections/FAQ.tsx
  카피: COPY.faq

  섹션 헤드: FAQs
  서브: 자주 묻는 질문들을 모았어요.
  추가 안내: 더 궁금한 점이 있으세요? [문의하기]

  ───────────────────────────────────────────────────────────────
  카테고리 1 — 도입 · 시작하기
  ───────────────────────────────────────────────────────────────

  Q1. 도입까지 얼마나 걸리나요?
    기본 세팅은 당일부터 가능합니다.
    ⏎⏎
    계정을 생성하면 상담 코치, 후기 안내, 케어 노트 기능을
    바로 확인하실 수 있습니다.
    ⏎⏎
    기존 환자 데이터를 옮겨와야 하는 경우에는
    →
    CSV·엑셀 파일 기준으로 보통 1~2일 내 정리가 가능합니다.

  Q2. 기존 데이터를 옮겨올 수 있나요?
    네, 옮겨오실 수 있어요.
    CSV·엑셀 파일을 그대로 올리시면 일괄 등록돼요.
    기존 CRM에서 내보낸 파일을 그 형식 그대로 받아요. 별도로 변환하실 일은 없으세요.

  Q3. 이미 쓰는 CRM이 있는데 같이 써도 되나요?
    네, 함께 사용하실 수 있습니다.
    ⏎⏎
    이음은 기존 CRM을 대체하기보다
    →
    실장님이 매일 반복하는 상담, 후기 안내, 진료 후 케어 업무를
    →
    보조하는 작업대에 가깝습니다.
    ⏎⏎
    기존 CRM의 환자 정보는 그대로 유지하시고,
    →
    실제 상담 운영 흐름만 이음에서 더 편하게 정리하실 수 있습니다.

  ───────────────────────────────────────────────────────────────
  카테고리 2 — 보안 · 데이터
  ───────────────────────────────────────────────────────────────

  Q4. 환자 정보가 AI에게 그대로 전달되나요?
    아니요. 그대로 전달되지 않아요.
    AI를 부르기 전에 환자분 성함·연락처·환자번호를 자동으로 가려서 보내요.
    AI 답변이 돌아오면, 다시 실제 환자분 정보로 자동 복원해드려요.
    ⏎⏎
    실장님은 평소처럼 환자분 성함으로 작업하시면 되고,
    AI 쪽으로는 식별 정보가 빠진 상태로만 전달됩니다.

  Q5. 우리 병원 데이터가 밖으로 나가지 않나요?
    나가지 않아요.
    국내 클라우드에 암호화해서 보관하고, 병원별로 따로 분리해서 관리해요.
    다른 병원이 우리 병원 데이터를 보는 일도, AI 학습에 쓰이는 일도 없어요.
    ⏎⏎
    AI에 전달되는 내용도 환자 식별 정보가 빠진 상태라,
    환자분 개인을 알아볼 수 있는 정보가 외부로 나가지 않습니다.

  ───────────────────────────────────────────────────────────────
  카테고리 3 — 의료법
  ───────────────────────────────────────────────────────────────

  Q6. 후기 기능이 의료법에 걸리지 않나요?
    이음은 의료법상 환자유인행위 이슈를 고려해
    →
    환자분이 직접 확인하고 게시하는 구조를 전제로 설계되었습니다.
    ⏎⏎
    AI는 상담 경험을 바탕으로 참고 초안을 제공하고,
    →
    병원 담당자가 검토한 뒤,
    →
    환자분이 본인 계정에서 직접 수정·게시하는 방식입니다.
    ⏎⏎
    다만 실제 운영 방식은 병원별 내부 기준과 관련 법령 검토에 따라
    →
    조정될 수 있습니다.

  ───────────────────────────────────────────────────────────────
  카테고리 4 — 이음에 대해
  ───────────────────────────────────────────────────────────────

  Q7. 이음은 어떤 도구인가요?
    성형·피부 클리닉 실장님의 상담 → 후기 → 케어가
    끊기지 않도록 AI가 이어주는 작업대예요.
    ⏎⏎
    기존 CRM이 환자 기록을 ‘저장’만 했다면,
    이음은 실장님이 매일 실제 업무를 ‘실행’하는 곳이에요.
    ⏎⏎
    원장님이 도입하시면, 실장님이 매일 쓰시고,
    그 결과를 환자분이 체감하시는 도구예요.
